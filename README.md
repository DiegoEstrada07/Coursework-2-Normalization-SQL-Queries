# Coursework-2-Normalization-SQL-Queries

Employee Database Normalization & SQL Queries
Assignment Overview

This project demonstrates the process of database normalization and SQL implementation using SQL.

The assignment begins with a raw employee dataset containing redundant information. The data is then normalized up to Third Normal Form (3NF) to eliminate redundancy and ensure data integrity.

After normalization, the database schema is implemented using SQL CREATE TABLE statements, followed by inserting sample data and writing SQL queries to retrieve and analyze the data.

Key Components of the Project

1. Normalization Process:

✪ Identified candidate keys and functional dependencies.

✪ Converted the dataset through 1NF, 2NF, and 3NF.

✪ Removed redundancy and transitive dependencies.

2. Database Schema Design:

Created two tables:

    ✪Department

    ✪Employee

Implemented primary keys, foreign keys, and constraints.

3. Data Population

Inserted 30+ records across the tables while maintaining referential integrity.

4. SQL Queries

Wrote SQL queries to perform:

   ✪ Aggregations (AVG, SUM, MIN, MAX)

   ✪ Filtering and pattern searches

   ✪ Joins between tables

   ✪ Grouping and HAVING conditions
